## For Users

### Requirements

- Overlord CLI (`ovld`) on `PATH`. See the Overlord docs / Settings ‚Üí Agents & MCP.
- Cursor agent binary (`agent`) on `PATH` for Overlord-launched sessions.
- `MISSION_ID` is exported automatically when a Cursor session is launched from Overlord.

### Install

From an OpenOverlord checkout:

```bash
ovld agent-setup cursor
```

This copies managed plugin files to `~/.cursor/plugins/local/overlord`, merges a `beforeSubmitPrompt` hook into `~/.cursor/hooks.json`, and adds a `Shell(ovld protocol:*)` permission allow rule to `~/.cursor/settings.json` when missing.

Re-run safely any time the connector contract version changes:

```bash
ovld agent-setup cursor
ovld doctor
```

### Slash commands

- `/attach` ‚Äî attach this session for mission execution
- `/connect` ‚Äî route this session onto an existing mission
- `/load` ‚Äî load mission context without attaching
- `/create` ‚Äî create a draft mission
- `/prompt` ‚Äî create and execute immediately
- `/discuss-objective` ‚Äî submit a draft objective for discussion
- `/add-objectives` ‚Äî append ordered objectives to a mission
- `/record-work` ‚Äî record already-completed work as a new mission
- `/spawn` ‚Äî deprecated alias for `/prompt`

### Namespaced components

Inside Cursor the plugin surfaces as:

- skill ‚Üí `overlord-mission`
- MCP server ‚Üí `overlord`
- rules ‚Üí `overlord-local`

Prompts generated by the runner should reference these names in bundle instruction mode.

## For Developers

### Overlord layer map

This connector is intentionally reviewable against the four connector layers in `connectors/README.md`:

- Connector Core: `../../core/overlord-mission/SKILL.md` and its `reference/` files carry the durable attach, update, ask, and deliver workflow.
- Connector Plugins: `skills/overlord-mission/SKILL.md`, `commands/`, `hooks/`, `rules/`, and `scripts/` extend the core for Cursor.
- Plugin Adapters: `.cursor-plugin/plugin.json` declares the Cursor-native plugin package.
- Prompt Wrappers: `prompt-wrapper.md` describes the launch-time prompt shape for Cursor.

### What ships

- `skills/overlord-mission/SKILL.md` ‚Äî Cursor-specific overlay bundled with the shared core reference files.
- `commands/*.md` ‚Äî slash commands for session routing, objective discussion, mission creation, and work recording.
- `hooks/overlord-user-prompt-submit.sh` ‚Äî `beforeSubmitPrompt` follow-up capture through `ovld protocol hook-event`.
- `rules/overlord-local.mdc` ‚Äî always-on workflow rules for Cursor sessions.
- `mcp.json` + `scripts/overlord-mcp.mjs` ‚Äî MCP bridge to common `ovld protocol` operations.
- `conformance-manifest.yaml` ‚Äî connector conformance declaration for the Overlord contract.
- `prompt-wrapper.md` ‚Äî Cursor launch guidance.

The shared Connector Core source is outside this adapter at `connectors/core/overlord-mission`. Setup copies the bundled skill and reference files into the installed plugin so the runtime package is self-contained.
