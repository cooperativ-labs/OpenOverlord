Integrations connect Overlord with external providers and agent tools. Configure
them from the workspace or project settings, then test the connection before
relying on it in an automated workflow.

The integration catalog and available permissions can vary by deployment. Only
enable the providers your team needs.

## Available integrations

- **[Everhour](#everhour)** — time tracking synced to your missions and projects.
- **[Webhooks](/webhooks/)** — send mission events to your own external systems.

## Everhour

Connect an Everhour account to track time directly from Overlord missions.

Everhour timers are per user, so each teammate adds their own API token.

1. **Get your API token.** In Everhour, open **Settings → My Profile** and copy
   the value under **API Token**.
2. **Save it in Overlord.** Open your account settings, paste the key into the
   Everhour section, and save. The key is stored per user with row-level
   security.
3. **Sync a project.** Open a mission in the target organization, and in the
   **Project** section choose **Sync Projects to Everhour**. Synced projects show
   an `(Everhour)` marker in the dropdown.
4. **(Optional) Share one Everhour project** across several Overlord projects so
   their time rolls up together — under a project's **Settings → Integrations**,
   link an Everhour project already connected to another Overlord project in the
   same organization.
5. **Verify.** Open any mission, start and stop a timer under **Time Tracking**,
   and confirm entries appear.

Notes:

- Timers are per user, and only one can run at a time — starting a timer on
  another mission stops the previous one.
- A mission must have a project assigned, and that project must be synced to
  Everhour.
