---
title: Local or Cloud
description: Choose where your Overlord workspace runs.
---

Overlord supports a local edition and a hosted Cloud edition. Both use the same work model and CLI surfaces.

## Local

The desktop app and backend run on your computer. This is useful when repositories and mission data should stay on the machine and you do not need a shared hosted workspace.

## Cloud

The backend is hosted so teammates and multiple machines can use the same workspace. Your repositories and agent execution still remain on the targets you configure.

Choose the backend during setup, or switch it from the CLI with `ovld config set local` or `ovld config set cloud <url>`.
