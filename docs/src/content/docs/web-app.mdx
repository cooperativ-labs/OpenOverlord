The web app is the shared control surface for workspaces, projects, missions,
activity, deliveries, and settings. Use it to create and review work from any
signed-in device.

The web app does not need access to a local checkout to display a mission.
Execution still happens on the target selected for the project.

## What it does

The web app is where you:

- create and edit missions and objectives
- organize work by project
- review mission activity and artifacts
- answer agent questions
- manage account settings, authentication, and connected agents

## Why it matters

The web app gives everyone a common view of the work without forcing them into a
terminal session or a specific agent UI. The board streams real-time updates as
agents work, and missions track read / unread state per reviewer so it is easy
to see what changed since you last looked.

## Common tasks

- turning a request into a mission
- watching live progress from an agent
- reviewing deliverables before approval
- checking account, authentication, and agent settings
